Configuración VirtualBox

Paso 1: Configurar Adaptadores de Red

Para cada máquina virtual, necesitas configurar dos adaptadores de red:

Adaptador 1 - Acceso a Internet (NAT)

Configuración: - Habilitar adaptador de red: ✓ - Conectado a: NAT - Nombre: (automático) - Avanzadas: Permitir todo el tráfico

Adaptador 2 - Red Interna

Configuración: - Habilitar adaptador de red: ✓ - Conectado a: Red interna - Nombre: internet - Avanzadas: Permitir todo el tráfico ⚠️ IMPORTANTE: El nombre "internet" debe ser EXACTAMENTE igual en las 3 máquinas virtuales.

Configuración por VM

Windows 7

Adaptador 1: NAT (Internet)
Adaptador 2: Red interna "internet"
IP estática: 192.168.1.6

Ubuntu Desktop

Adaptador 1: NAT (Internet)
Adaptador 2: Red interna "internet"
IP estática: 192.168.1.7

Ubuntu Server

Adaptador 1: NAT (Internet)
Adaptador 2: Red interna "internet"
IP estática: 192.168.1.8

⚠️ Notas Importantes

  • El nombre de la red interna debe ser idéntico en todas las VMs
  • Cada VM necesita una IP única en el rango 192.168.1.x
  • La máscara de subred será 255.255.255.0 para todas
  • No configurar gateway en la red interna para evitar conflictos de rutas

Configuración Windows 7

Paso 1: Acceder al Centro de Redes

Panel de Control → Centro de redes y recursos compartidos

Paso 2: Configurar Adaptador de Red Interna

En el Centro de redes verás dos conexiones:

  • Conexión de área local 1: Acceso a Internet (NAT)
  • Conexión de área local 2: Red interna (a configurar)
Pasos: 1. Click derecho en "Conexión de área local 2" 2. Seleccionar "Propiedades" 3. Doble click en "Protocolo de Internet versión 4 (TCP/IPv4)" 4. Seleccionar "Usar la siguiente dirección IP" Configuración: - Dirección IP: 192.168.1.6 - Máscara de subred: 255.255.255.0 - Puerta de enlace: (dejar vacío) - Servidor DNS preferido: 8.8.8.8 - Servidor DNS alternativo: 8.8.4.4 5. Click "Aceptar" → "Aceptar"

Paso 3: Habilitar PING en Windows Firewall

Por defecto, Windows 7 bloquea las solicitudes PING. Para habilitarlas:

1. Panel de Control → Sistema y seguridad → Firewall de Windows 2. Click en "Configuración avanzada" (lado izquierdo) 3. En "Reglas de entrada" → Click derecho → "Nueva regla" 4. Tipo de regla: "Personalizada" → Siguiente 5. Programa: "Todos los programas" → Siguiente 6. Protocolo: "ICMPv4" → Siguiente 7. Ámbito: "Cualquier dirección IP" → Siguiente 8. Acción: "Permitir la conexión" → Siguiente 9. Perfil: Marcar todas las casillas → Siguiente 10. Nombre: "Permitir PING" → Finalizar

💡 Verificar Configuración

# Abrir símbolo del sistema (cmd) ipconfig # Debería mostrar: Adaptador de LAN 2: Dirección IPv4: 192.168.1.6 Máscara de subred: 255.255.255.0

Solución de Problemas

Si no puedes hacer ping:

  • Verifica que la regla de firewall esté habilitada
  • Reinicia la máquina virtual después de los cambios
  • Confirma que el adaptador 2 esté conectado a "Red interna"
  • Verifica que el nombre de red sea exactamente "internet"

Ubuntu Desktop 20.04

Configuración con NetworkManager

Ubuntu Desktop utiliza una interfaz gráfica simple para configurar la red:

Pasos: 1. Click en el icono de red (parte superior derecha) 2. Seleccionar "Configuración de Red" o "Network Settings" 3. Verás dos adaptadores: - Adaptador cableado 1: Internet (DHCP automático) - Adaptador cableado 2: Red interna (a configurar) 4. Click en el ⚙️ (configuración) del Adaptador 2 5. Ir a la pestaña "IPv4" 6. Cambiar método de "Automático (DHCP)" a "Manual"

Configuración IP Manual

Dirección: 192.168.1.7 Máscara de red: 255.255.255.0 Puerta de enlace: (dejar vacío) DNS: 8.8.8.8, 8.8.4.4 Click "Aplicar" y reiniciar la máquina virtual

Verificación desde Terminal

# Verificar configuración de red ip addr show # Debería mostrar algo como: 3: enp0s8: <BROADCAST,MULTICAST,UP,LOWER_UP> inet 192.168.1.7/24 brd 192.168.1.255 scope global enp0s8 # Verificar conectividad ping 192.168.1.6 # Ping a Windows 7 ping 192.168.1.8 # Ping a Ubuntu Server

Ubuntu Server 20.04

Identificar Adaptadores de Red

# Listar interfaces de red ip a # Salida típica: 1: lo: <LOOPBACK,UP,LOWER_UP> (loopback) 2: enp0s3: <BROADCAST,MULTICAST,UP,LOWER_UP> (Internet - NAT) 3: enp0s8: <BROADCAST,MULTICAST,UP,LOWER_UP> (Red interna)

Configuración con Netplan

Ubuntu Server 20.04 utiliza Netplan para configurar la red:

# Editar configuración de Netplan sudo nano /etc/netplan/00-installer-config.yaml

⚠️ Sintaxis YAML Crítica

La sintaxis en Netplan debe ser perfecta:

  • 2 espacios por cada nivel de indentación
  • NO usar tabulaciones, solo espacios
  • Cualquier error de sintaxis causará fallos
# Archivo: /etc/netplan/00-installer-config.yaml network: ethernets: enp0s3: dhcp4: true enp0s8: addresses: - 192.168.1.8/24 # gateway4: (comentado para evitar conflictos de ruta) nameservers: addresses: - 8.8.8.8 - 8.8.4.4 version: 2

Aplicar Configuración

# Verificar sintaxis (no debe mostrar errores) sudo netplan try # Si no hay errores, aplicar cambios sudo netplan apply # Verificar configuración ip addr show enp0s8

Verificar Rutas por Defecto

# Ver ruta por defecto ip route show | grep default # Salida esperada: default via 10.0.2.2 dev enp0s3 # Esto confirma que enp0s3 (NAT) es la ruta por defecto # para acceso a Internet, no enp0s8 (red interna)

Pruebas de Conectividad

Comandos de prueba:

# Ping a otras VMs en la red interna ping 192.168.1.6 # Windows 7 ping 192.168.1.7 # Ubuntu Desktop # Verificar acceso a Internet (por enp0s3) ping google.com # Ver tabla de rutas completa route -n # Verificar servicios SSH (si aplicable) sudo systemctl status ssh

Solución de Problemas Netplan

Errores comunes y soluciones:

  • Error de sintaxis YAML: Verificar espacios (no tabs)
  • Sin conectividad: Verificar nombres de interfaces con ip a
  • Conflicto de rutas: Comentar gateway4 en red interna
  • Cambios no aplicados: Reiniciar VM después de netplan apply

Resumen Ejecutivo

Este proyecto presenta una innovación arquitectónica fundamental en el campo de la computación médica, introduciendo el concepto de "sólido geométrico" para la coordinación de sistemas multi-agente. La esencia del proyecto es el desarrollo de una solución de software arquitectónica de vanguardia para el sector de la salud, enfocada en la escalabilidad, la especialización de agentes y la robustez.

La arquitectura está concebida para integrar de manera sinérgica agentes de inteligencia artificial y software automatizado, soportando aplicaciones médicas críticas desde el diagnóstico hasta la investigación de frontera.

El Concepto de Sólido Geométrico

Representación conceptual del sólido geométrico multi-agente

La arquitectura de sólido geométrico representa un paradigma arquitectónico con base matemática sólida en la teoría de grafos y la topología algebraica.

  • Vértices: Cada máquina virtual (Vi) representa un vértice en un espacio n-dimensional
  • Aristas: Interconectividad definida mediante matriz de adyacencia Aij
  • Caras: Simbolizan la colaboración operativa entre múltiples agentes
Capacidad Computacional Emergente

Ctotal = Σ Ci + α · Σ Aij · S(Vi, Vj)

Donde:

  • Ci: Capacidad individual de la VM i
  • α: Factor de sinergia (0.15 - 0.3 para sistemas médicos)
  • S(Vi, Vj): Función de sinergia entre agentes

#!/bin/bash # Script completo de diagnóstico para múltiples VMs # Nombre: vm_network_diagnostics.sh LOG_FILE="/tmp/vm_diagnostics_$(date +%Y%m%d_%H%M%S).log" log() { echo "[$(date '+%Y-%m-%d %H:%M:%S')] $1" | tee -a "$LOG_FILE" } test_connectivity() { local ip=$1 local name=$2 log "Testing connectivity to $name ($ip)..." if ping -c 3 -W 5 "$ip" &>/dev/null; then log "✓ $name is reachable" # Test SSH if port 22 is open if nc -z "$ip" 22 2>/dev/null; then log "✓ SSH service available on $name" fi # Test HTTP if port 80 is open if nc -z "$ip" 80 2>/dev/null; then log "✓ HTTP service available on $name" fi else log "✗ $name is NOT reachable" fi } # Array de VMs a testear (personalizar según tu configuración) declare -A VMS=( ["192.168.1.6"]="Windows-7" ["192.168.1.7"]="Ubuntu-Desktop" ["192.168.1.8"]="Ubuntu-Server" ["192.168.1.10"]="Load-Balancer" ["192.168.1.20"]="Database-Master" ) log "=== VM Network Diagnostics Started ===" log "Network interfaces:" ip addr show | grep -E "(enp0s|eth)" >> "$LOG_FILE" log "Routing table:" ip route show >> "$LOG_FILE" for ip in "${!VMS[@]}"; do test_connectivity "$ip" "${VMS[$ip]}" done log "=== Diagnostics completed. Log saved to: $LOG_FILE ==="
